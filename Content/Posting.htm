Posting

Posting is the process in RouteMatch that makes scheduled trips available for prossessing during your agency's regular service. Many agencies configure RouteMatch to execute posting automatically when the software is opened. The Scheduling module provides the following features to help your agency do the following:

  • Test posting success
  • Correct posting failures

How to Test Posting Success

  1. If RouteMatch encounters a problem when auto-posting, an error displays when you log into the application. The error message contains a list of the specific items that failed to post and the reason(s) why.

All RouteMatch users at your agency will see this warning message when they log in until the posting issues are resolved.

  1. You need to enter your password again on this screen to acknowledge the posting failure, and then click the Login button. You can also click the Copy to Clickboard button to copy/paste the information for future reference.
  2. The system then automatically opens the Scheduling module. Click the View button on the toolbar and select the Posting History option.

  1. Notice that the screen that appears provides a calendar view that displays the posting errors. Only the trips causing the errors remain unposted.
  2. From here, you can either allow the system to correct the errors automatically (according to the displayed method), or manually correct the error yourself. This screen also allows you to do the following:

  • View which users "acknowledged" the posting failure and when they did it

  • Manually post vehicles even if you have auto-posting enabled
  1. Correct the errors using the manual or automatic methods.
  2. Once all of the errors are corrected, you should be able to successfully post the full schedule.
