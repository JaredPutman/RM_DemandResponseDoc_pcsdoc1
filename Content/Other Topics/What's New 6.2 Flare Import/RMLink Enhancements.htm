RMLink Enhancements

The following features are new to the RMLink Message Center starting in #6.2.

Increased Character Send Text Limit

A new variable has been added to the GPS section in the settings module, RMLink sending message length limit. The default value is set to 250 characters in length.

When typing a message in the dispatching send message box, the maximum characters allowed is based on this setting. If there is no number or the setting is not available, the default value is 120 characters.

 

Different devices have different limitations on the amount of text that can be viewed.

Format and Filter

The RMLink Message Center now allows you to format, filter and create custom dataviews. This new look matches other grids found in the application. When editing the dataview, you can now change color priorities in the formatting tab or add how messages are filtered in the filtering tab. In addition, checkboxes have replaced the “*” indication of a read message in the message center grid. The types, message or command, are also shown as text in the Kind column (MSG or CMD).

Send Messages to Vehicle Pools

Now you can send a message to all vehicles belonging to a vehicle pool, either individually or as group. This is selected from the toolbar grid for Active Devices.

Edit Canned Message text

Dispatchers are now able to edit text of canned messages from the Send Message window in the dispatching module. When they select a canned message from the drop-down, its text displays in the box below. The dispatcher can then make changes to this text.

Filter by Run

You can now filter the messages by their corresponding runs by opening the Run tab in the Message Filter pane.
