Verification Module Tool Bar

Item

Description

Date

Select a date to verify from the drop-down calendar. Trips must have already been completed before they can be verified so this must be a date in the past (yesterday or earlier).

 Save Run

Save the selected run information. There are several options for saving:

  • Save With Validation – Save and validate all the information for the selected run. This marks the vehicle as “verified” in the drop-down menu.
  • Save Without Validation – Save the changes you have made, but do not finalize the changes with a validation.
  • Save With Validation And Analysis— Saves and validates all information for a selected run and marks the vehicle as “verified.” A re-analysis of the validated information is also performed.
  • Save Without Validation But With Analysis— Saves any changes made and re-analyzes the current vehicle.  Re-analyzing vehicle data is important as additional changes may alter the number of exceptions found and displayed in the analysis notes column.

 Calculate Costs

Calculate the trip cost for a selected vehicle, a selected date range, or selected trips. Vehicles and trips can be verified or unverified.

 Reassign Trips

Open the Assign Trips screen to assign trips to different runs or vehicles.

 Move Run to Vehicle

Open the Assign Runs/Breaks to Vehicle screen to assign the selected run to a different vehicle. This button is only active when a named run is selected.

 Run Management

Open the Run/Driver Management screen to assign runs to vehicles and create unscheduled runs. You can also use this screen to assign drivers to runs and vehicles.

 Breaks

Open the Break Verification screen to add and verify any breaks the driver took.

 New Trip

Create a new trip. You might need to do this if you know a specific trip was taken but it does not appear in verification.

Customer Trip History

Open the Customer Trip History screen and view the trips a customer has taken in the past for a selected date range.

 Show Trip in Trips

Open the selected trip in the Trips module.

 Delete Trip

Delete the selected trip.

 Stop Mode

View the Verification grid by individual trips or by each stop (Pickup and Drop-off).

 Print

Print Runs by Date - View and print the Runs by Date report

Print Trip Audit Log Day Of – This allows you to quickly view changes made to the trip. Clicking this item brings you to the trip to view the log of changes. This displays when you have selected a trip in the verification data view.

 Grid Settings

Open the Verification Fields screen and select which trip detail fields will appear on the Verification grid.

 Save Grid Settings

Save any changes you make to the Verification grid. This feature is active if the Auto-Save box is unchecked in the Verification Fields dialog.

 

 

 Search

Allows you to search for various types of information in the Verification Module.

 Show Date / Summary Info

Displays the date row in the run summary area of the screen and the date in all the time columns (Stop Time, PU Time, and DO Time) of the Verification grid.

The Summary Info shows a summary of the vehicle grid information for the selected date and / or run. The summary is shown below the Trip Detail Information bar.

 Analyze Current Vehicle

This feature allows you to quickly examine runs to determine if there are discrepancies within individual odometer readings or recorded trip stop times in the Analysis Note column. These discrepancies are shown in red.

 Accept Suggested value for current run

Allows you to accept the results shown on the Analyze Current Vehicle grid. Once the results are accepted, the text in the grid will turn from red to black. The suggested values are based on travel times and distances computed by the RM Scheduling engine.

 Analyze One Day

This feature performs an analysis of all the vehicles and trips for a selected date.
