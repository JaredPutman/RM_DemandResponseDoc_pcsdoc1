Compute Cost Criteria

Define the criteria that determine which trips, funding sources, etc. apply to this billing rule. You can either select a previously created set of criteria from the drop-down list, or create a new set by completing the following steps:

  1. Click the Create New Criteria button to open the Criteria Editor screen.

  1. From here, enter a Name for this criteria set (Required).
  2. Enter a Description for this criteria set. What you enter here displays in the Description tab of the cost criteria section of the Billing screen, so it should explain the criteria’s intended use.
  3. In the Kind field, select a condition that must be met for these criteria to apply. For example, these criteria can be set to apply to verified trips, dispatched trips, etc.
  4. Click the New button to open the Criteria Row Editor. The criteria you create here allow you to define which trips and runs apply to this billing rule.

  1. On the Criteria Row Editor screen, select to use the standard or advanced mode of this screen. The process of creating criteria in advanced mode is covered later in this chapter.
  2. Select a parameter from the list on the left. For runs, there are only four choices on this list.
  3. Specify how this parameter compares with its value from the drop-down list. For example, the chosen parameter can be equal to, greater than, less than, etc. in relation to the entered value.
  4. Determine whether this is a constant value, global value, or SQL language.
  5. Enter or select a value for this parameter.
  6. Click Save and Continue to create more rows.
  7. When all of your needed criteria have been set. Click Save and Return to close the window and return to the Billing module.

 

Make sure that criteria do not overlap. If they do overlap, an error displays in the Verification module and the customer pay on the manifest is $99,999.99.

 How to Use the Advanced Option to Create Criteria

If you use the Advanced option, you must enter SQL language to create the criteria from scratch. You should attempt this only if you are familiar with Microsoft SQL and can use the language correctly. Contact RouteMatch if you require assistance.

  1. After you click the Create New Row button and open the Criteria Row Editor screen, select the Advanced option.
  2. On the Advanced screen, enter the SQL language for the new criterion.
  3. Click the blue check-mark button to verify the SQL statement.
  4. Save the criteria when you have entered all the necessary information. You do have the option to combine the criteria you create under the advanced option with any criteria created under the standard option.

Example

Create a criteria with the following: Funding Source = Medicaid

Then, switch over to the Advanced option and enter the following SQL language:

(DropOffAddress IN (‘10375 E HARVARD AVE’,’ 9900 E ILIFF AVE’) OR PickUpAddress IN (‘10375 E HARVARD AVE’,’ 9900 E ILIFF AVE’))

This creates cost criteria that applies to trips with the Medicaid funding source and has a pickup or a dropoff at 10375 East Harvard Avenue.
