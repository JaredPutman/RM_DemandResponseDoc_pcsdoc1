Compute Non-Customer Pay Cost

These fields determine how much a customer’s funding source is expected to pay when they take a trip under this billing rule.

  1. Select an option for how the system should compute non-customer pay cost. The selection you make here determines the parameters you then need to define in the grid below. You have the following cost computing options:
  • Flat Fee Only – Charge a fixed amount for trips regardless of the miles or length of time traveled.
  • Flat Fee Only (Full Control) – This choice has the same parameters as the Flat Fee Only option, but you can override the corresponding funding sources’ Bill Attendant, Bill Guest, and Bill No Show settings.
  • Individual Miles and Explicit Time Interval – The amount charged for this billing rule is based on the miles and the exact length of time (down to minutes) traveled.
  • Individual Miles and Explicit Time Interval (Full Control) – This choice has the same parameters as the Individual Miles and Explicit Time Interval option, but you can override the corresponding funding sources’ Bill Attendant, Bill Guest, and Bill No Show settings.
  • Individual Miles and Hours – The amount charged for this billing rule is based on the miles and hours traveled.
  • Individual Miles and Hours (Full Control) – This choice has the same parameters as the Individual Miles and Hours option, but you can override the corresponding funding sources’ Bill Attendant, Bill Guest, and Bill No Show settings.
  • Individual Miles and Hours with Deadhead and Wait Time – The amount charged for this billing rule is based on the miles and hours traveled. This option also allows you to apply extra charges for deadhead and wait time.
  • Individual Miles with Independent Guest and Attendant Charges – The amount charged for this billing rule is based on the miles traveled. This option also allows to apply extra charges to guests and attendants.
  • Shared Ride – All passengers share the cost of the trip, which is based on the miles traveled. This option also allows for extra charges to be applied to guests and attendants.
  • Master Trip Cost Function – This billing rule is accessible from the Master Cost Billing Rule Attributes screen.
  • Shared Miles – All the passengers share the cost of the trip, which is based on the miles traveled.
  • Shared Miles and Hours – All passengers share the cost of the trip, which is based on the miles and hours traveled.
  • Shared Miles and Hours (Full Control) – This choice has the same parameters as Shared Miles and Hours, but you can override the corresponding funding sources’ Bill Attendant, Bill Guest, and Bill No Show settings.
  • Run Charge – This option allows you to define the various charges that can apply to runs. For example, the run fee, revenue hours/miles, service hours/miles, etc. can all be defined here. When creating a billing rule for runs, this is the only selectable option.

The Description tab displays a definition of the computation and its corresponding parameters.

 

If you select an option that is not Full Control, the corresponding funding source must have the Bill Attendants, Bill Guest, and Bill No Show options defined.

  1. In the How Cost Computed field, select a method for computing the non-customer pay cost. This information displays in the Verification module:
  • Use Current Billing Rule Group – The non-customer pay cost is calculated using the name of the billing rule’s group.
  • Use Current Billing Rule Name – The non-customer pay cost is calculated using the name of the billing rule’s name.
  • Use Trip’s Funding Source and Override – The non-customer pay cost is calculated using the name of the billing rule’s funding source and override.
  1. Define the parameters of how this billing rule computes non-customer pay cost. The parameters appear in the Detail tab, which contains the following columns:

  • Parameter Name – The name of the parameter associated with the selected method for computing cost.
  • Parameter Type – The parameter’s corresponding classification.
  • Value Kind – The data type associated with the parameter’s value. The available options are fully explained in the next section.
  • Value – The definition of the parameter. For example, Flat Fee = 5.00 means that the flat fee for this billing rule is $5.00.
  1. For each parameter, select a value kind and define it’s value.
Value Kinds
Value Kind

Definition

Column

The value is drawn from your RouteMatch database. Enter the name of the specific table and column of the field from the Trips module that contains this information.

These values must follow this format: <TableName>.<ColumnName>

For example: tblTripsActual.Cancelled

Constant

The value applies to all parts of its assigned parameter. For example, you can use this option to apply a $3.00 charge to every trip that falls under this billing rule.

SQL

The value is defined with database-specific language.

Local Value

The value is valid only within the scope of the corresponding parameter. Click the ellipsis (…) to define the value and the corresponding date range of when the parameter applies.

Global Value

The value is a pre-defined option created in the Global Parameter Manager screen. Global Values can be used throughout the Billing module. The process for creating global values is explained later in this guide.

System

The value is chosen from a list of options that is predefined in RouteMatch.

Alias Column

The value is made up of a computation of a group of columns from the Trips module.
