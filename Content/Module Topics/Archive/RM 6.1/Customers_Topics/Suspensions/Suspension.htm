Suspension

The Suspension tab allows you to suspend a customer’s service for a period of time based on a predetermined reason. You will not be able to create trips for a suspended customer and the system deletes any existing trips that fall in the suspension period. You can also manage Customer abuses in this tab.

The Suspension tab automatically sends messages to the Customer Communications module to deliver letters of warning and suspension to your customers when abuse points reach certain thresholds. Customers will receive a Warning and suspension letter when a customer’s abuse points match a global database parameter that the System Administrator can set or change. The warning and suspension global parameters cannot be set to the same value, or set to zero. Additionally, the suspension value cannot be set below the warning value.

Note

In order to use the Suspension Tab, an administrator must set customer Suspension Reasons.
