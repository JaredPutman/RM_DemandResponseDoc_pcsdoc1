Service Settings

Warning

The configurations you make with these settings can have a significant effect on RSE. Please contact RouteMatch if you require assistance.

Field Description

Trip Length Restriction Type

Determine how the system should limit the length of time a customer can spend on a trip. There are two options:

  • By Specific Time – Limit the trip length by a specific amount of minutes. This option will activate the Maximum Trip Time field.
  • By Adjusted Travel Time – Limit the trip length by applying a multiplier. This option will activate the Trip Length Factor field.

Maximum Trip Time (In Minutes)

If you decide to use the By Specific Time option for the trip length restriction, then define the maximum number of minutes a customer can spend on a trip.

For example, if you enter "90" into this field, customers using this service cannot have trip lengths exceeding this 1.5 hour window.

Trip Length Factor

If you decide to use the By Adjusted Travel Time option for the trip length restriction, enter a multiplier in this field that applies to the calculated trip length to define the maximum amount of time a customer can spend on a trip.

For example, if a customer’s calculated trip length is 15 minutes, and this option is set to 3, the customer’s trip cannot exceed 45 minutes (15 x 3=45).

Allow Scheduling to Runs Using Optimizer

Allow RSE to schedule this service’s trips to runs. This setting has two options:

  • Yes – Include runs for optimization.
  • No – Do not include runs for optimization.

Default Negotiated Request PU Time Window (Min. Before)

The number of minutes before a customer’s negotiated time that RSE is allowed to schedule their pickup.

Default Negotiated Request PU Time Window (Min. After)

The number of minutes after a customer’s negotiated time that RSE is allowed to schedule their pickup.

Default Negotiated Request DO Time Window (Min. Before)

The number of minutes before a customer’s negotiated time that RSE is allowed to schedule their drop-off.

Default Negotiated Request DO Time Window (Min. After)

The number of minutes after a customer’s negotiated time that RSE is allowed to schedule their drop-off.

Maintain Initial and Negotiated Request Times Separately

If your agency negotiates trip times with customers, set this to Yes to have the system keep track of both the customer’s requested and negotiated trip times.

Default Initial Request PU Time Window (Min. Before)

The number of minutes before a customer’s requested time that RSE is allowed to schedule their pickup.

Default Initial Request PU Time Window (Min. After)

The number of minutes after a customer’s requested time that RSE is allowed to schedule their pickup.

Default Initial Request DO Time Window (Min. Before)

The number of minutes before a customer’s requested time that RSE is allowed to schedule their drop-off.

Default Initial Request DO Time Window (Min. After)

The number of minutes after a customer’s requested time that RSE is allowed to schedule their drop-off.

Trip Booking Horizon Limit (in Days)

The number of days in advance to allow a customer to book a trip. Configuring this to 0 leaves the limit open-ended so customers can book trips at any time.

Trip Booking Horizon Limit Requires Supervisory Override

If set to yes, an administrator must enter their password to approve a customer’s trip if they are requesting it outside the booking horizon limit.

ADA Corridor Required

Select whether an ADA corridor applies to the trips of this service.

ADA Corridor Requires Supervisory Override

If set to yes, an administrator must approve a customer’s trip if it will take place outside the ADA corridor.

Timing Preference(s)

Click the ellipsis and select the various timing preferences that apply to this service in the Trips module. This setting has five options:

  • Drop-off – The customer has a specific time when they need to be dropped off, but their pickup time is flexible.
  • Pickup – The customer has a specific time when they need to be picked up, but their drop-off time is flexible.
  • Pickup With Drop-off at Appointment Time – The customer has a specific time when they need to be picked up and dropped off in order to make an appointment. Use this option only when it is necessary as the extra requirements give RSE fewer options for scheduling the trip.
  • Drop-off Computing Pickup Time – Select this preference when negotiating trip times to define the customer’s desired drop-off time and have RouteMatch automatically calculate their pickup time. Use this option only when it is necessary as the extra requirements give RSE fewer options for scheduling the trip.
  • Pickup Computing Drop-off Time – Select this preference when negotiating trip times to define the customer’s desired pickup time and have RouteMatch automatically calculate their drop-off time. Use this option only when it is necessary as the extra requirements give RSE fewer options for scheduling the trip.
