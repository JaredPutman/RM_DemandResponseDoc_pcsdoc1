Post IVR Callouts

This option queues the messages for the IVR system to call. Once the trips are queued up for the IVR calling system, a pop up window displays reporting how many trips were posted.

Post trips to Dispatching before posting IVR Callouts.

A feedback message displays indicating the number of callout requests that posted along with the start time, end time, and the maximum number of retries.

The RMXS message agent takes the "message body" from RouteMatch and adds other information to it. The settings established in RouteMatch do not affect RMXS settings. The RouteMatch message delivery requests contain all necessary delivery parameters for the IVR callout, including when the process to fill the RMXS queue table will occur. The actual calling begins per the RMXS properties file.
