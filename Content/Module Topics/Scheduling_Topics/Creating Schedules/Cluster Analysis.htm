Cluster Analysis

The Cluster Analysis feature allows you to quickly identify trips that have the same pickup and drop-off address called Cluster Analysis. These clusters would normally be scheduled on the same vehicle however now you can manually remove trips from clusters or add trips to clusters if you find it appropriate to do so.

To use this feature, you must have the semiclustering settings enabled under the profile that you are using for cluster analysis. These settings are highlighted below. Enabling semiclustering is required, however allowing for multiple services in semiclustering is optional (those items are unchecked and highlighted in yellow).

How to run Cluster Analysis

  1. Click Cluster Analysis from the drop-down Optimization menu in Scheduling.

  1. In the popup window, select the desired vehicle pools and profile. Click the Run button.

  1. The results appear in the new window which you can drag to another monitor and keep open while working in the main application window. You can also expand the clusters and view trips assigned to each vehicle. Right-clicking on a specific trip, you can navigate to the scheduled trips view in the scheduling module.
