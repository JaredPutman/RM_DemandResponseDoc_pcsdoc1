How to Execute a Schedule Optimization

  1. Click the Optimize button on the Scheduling Module and select Run Optimization Engine: RSE™ to open the Select Vehicles/Run To Optimize window. This window lists the available vehicles and runs on the left side of the screen.
  2. At the top of the screen, select whether you want to restrict the optimization according to the grouping currently selected in the Unscheduled Trips window. For example, you can use this to optimize only the trips under a specific mobility type.

  1. Select the vehicle or run from the tree on the left side. You can click the plus sign icon next to the vehicle to expand the list if there are runs already added to the vehicle.
  2. Click Add to move the vehicle or run to the optimize list. If you want to optimize all available vehicles and runs, click Add All. If you need to remove a vehicle from the optimize list, select the vehicle and click Remove, or click Remove All to remove all vehicles and runs from the list.
  3. Select an optimization profile from the drop-down list. Profiles determine which set of optimization tuning parameters RSE uses for the optimization.

Important Note about Profiles

Previously, RouteMatch provided for only one set of RSE optimization tuning parameters. Any changes to these parameters replaced the existing values, making it difficult to go back to the original settings. In 5.3.5, these settings were updated and can now be defined with multiple profiles allowing the user to switch back and forth between different setups.

This gives you the ability to modify RSE settings and keep the option to revert back to their original values. Profiles also allow you to apply multiple sets of values to different parts of scheduling optimization. For example, the user could apply one profile that highly favors vehicle efficiency to one set of vehicles, and then apply another profile that is less restrictive for another set of vehicles.

Optimizing Without RSE Profiles

If you do not want to use this feature, you can optimize with and make changes to the Master Profile. This is the profile created from your existing settings when your system is upgraded, or created with ‘factory settings’ when first installed.

  1. Select the optimization method from the drop-down at the bottom of the window. You have the following options:

Note

If you have trips that are set as frozen trips, none of the optimization methods will have any effect on them.

  • Adjust scheduled trip times, maintaining stop order - Selecting this option will optimize the trips currently scheduled on the selected vehicles while maintaining the stop order of the trips on the vehicle. New trips are not added and existing trips are not removed from the selected vehicles.
  • Reorganize schedule, removing conflicting trips - Selecting this option will optimize the trips currently scheduled on the selected vehicles. The vehicles may have their stop order changed, conflicting trips removed, and no new trips added to the selected vehicles.
  • Remove current schedule and reoptimize - Selecting this option will remove any scheduled trips that are not on a run from their assigned vehicles and attempt to determine the most efficient use of the trips. This option will also attempt to add unscheduled trips to the selected vehicles.
  • Insert unscheduled trips where possible - Selecting this option will attempt to insert unscheduled trips on to the selected vehicles. This option will not make changes to the trips currently scheduled to the selected vehicles.
  • Advanced - Selecting the Advanced option displays more optimization options. The Advanced options are shown below with a brief description and you may select multiple options for the optimization.

  • Keep scheduled trips on same vehicles - Selecting this option will optimize the trips on the vehicles and not add any new trips to the vehicles, unless you also select "Attempt to insert unscheduled trips." All trips currently scheduled on the selected vehicles will remain on those vehicles, but possibly not at the same time as they were originally.
  • Keep scheduled trips in same order – Select this option to specify if trips already on a vehicle, should be kept in the same order as previously scheduled. The combination of Keep scheduled trips on same vehicles, Keep scheduled trips in same order, and Keep scheduled trips at same time allow schedulers to choose more detail in how RSE should handle trips already scheduled to vehicles. We recommend that you not use this option so that RSE has the freedom to re-order trips on a vehicle to create a better schedule.
  • Keep scheduled trips at same time - Selecting this option will optimize the trips by preserving the times of the already scheduled trips. The Keep scheduled trips on same vehicles must also be selected to use this option.
  • Only trips scheduled by RSE – For each of the Keep scheduled trips… options, you can select this to only adhere to those restrictions if RSE scheduled the trips as opposed to the trips scheduled manually.
  • Allow trips on any service - Selecting this option will optimize the trips and allow trips to be scheduled on any selected vehicle regardless of service associated to the vehicle.
  • Attempt to insert unscheduled trips - Selecting this option will optimize the trips and attempt to insert unscheduled trips to vehicles where applicable.
  • Schedule trips within runs - Selecting this option will schedule trips within the selected run time windows. No trips will be scheduled outside of the defined run times.
  • Schedule trips around runs - Selecting this option will schedule trips around runs and prevent trips from being scheduled within runs.
  • Adjust Run Times Around Trips – When trips are added to a run, the time of the run is adjusted by the trips it contains. This keeps a large time gap from existing before and after the first and last trips. For example, if the run is scheduled to run between 9 AM and 5 PM and the trips scheduled to the run are between 11 AM and 3 PM, the run time will be adjusted according to the Adjusted Runs Include Travel Times option. If the travel time is 10 minutes for both the first pickup and last drop-off, the new run would be 10:50 AM to 3:10 PM.
  • Adjusted Runs Include Travel Times – Select this option to have RSE take into account travel time to and from the garage when adjusting the run time around trips.
  1. Click the Optimize button.
  2. The Optimization Results screen appears and has three panes: Optimized Vehicles, Current Schedule, and Proposed Schedule. The status of each trip on the vehicle is indicated by a symbol preceding the trip. The legend at the bottom of the Optimization Results screen explains the symbols. Notice that two trips were added to the schedule in the example below.

  1. Click Accept Vehicle to accept the schedule on the selected vehicle. Click Accept Schedule to accept the schedules on all vehicles, or click Cancel and Return to discard the schedule.
  2. Click Apply Schedule to finish the process and close the screen.
