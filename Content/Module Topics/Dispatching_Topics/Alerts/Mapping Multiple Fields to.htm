Mapping Multiple Fields to Trigger the Same Alert

In addition to setting basic alerts where a field change turns on an alert, you can define field changes to turn off an alert and you can define multiple fields to turn on or off the same alert. When multiple fields are mapped to the same alert, you must define an order in which the changes would be applied. The highest number in the order is applied last. Higher order changes override lower order changes. For example, you could map changes to both Guest Count and Attendants to an alert named Rider Count Change. If either field is changed, this alert is turned on. You could further define a change of vehicle to turn this alert off. So, if you changed the number of riders and the vehicle, the order would determine if the alert was turned on or off. If the vehicle change has the highest order, the alert would be off.

You can also format on values in descending order. For example, if a trip is less than 30 minutes from now make it blue, if it is less than 20 minutes make it yellow, if less than 10 make it red. In this case, you must list the formatting conditions in the proper order. Starting from the top of the list and moving down, the first format is the one that will apply to a row. Below are two examples: the one on the left using greater than and the one on the right using less than. Notice that the order is different.

Guest Count > 3  Red             Trip Time < 10  Red

Guest Count > 2  Yellow            Trip Time < 20  Yellow

Guest Count > 1  Blue            Trip Time < 30  Blue
