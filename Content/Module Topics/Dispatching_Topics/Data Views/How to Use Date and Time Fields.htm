How to Use Date and Time Fields to Build Filters

The date and time at which a trip becomes visible in Dispatching is designated as the trip being "posted". This posted date/time is stored in the Entered Date field, and this field can be added to the grid display for any data view. This field is useful for detecting when a trip has been posted to Dispatching. For example, if an agency posts trips at 5:00 a.m. each day, a filter of all trips with an entered date after 5:00 a.m. would show all trips added since the posting process executed. By using the additional conditions explained below, you can filter on trips added in the last hour, last 30 minutes, etc.

The following fields have additional date-related conditions for building filters:

  • StopDateTime
  • EnteredDate
  • RequestTime
  • TripDate (Scheduled Object Date)

The following fields have time-related conditions for building filters:

  • StopDateTime
  • EnteredDate
  • RequestTime

With the ability for a trip to span midnight, Dispatching may contain trips from different dates. The following additional date selectors allow you to see anything with a date of X days ago, X days in the future, or today:

  • "Days Before Today Equals"
  • "Days After Today Equals"
  • "Is Today"

The following additional time selectors allow you to look at windows of time before and after the time considered "now":

  • "Minutes Before Now More Than"
  • "Minutes Before Now Less Than"
  • "Minutes After Now More Than"
  • "Minutes After Now Less Than"
  • "Is Now"

So, for example, you can select to filter on the "StopDateTime" and select "Minutes After Now Less Than" and enter 30 in the value field. This would select all trips with a StopDateTime in the next 30 minutes. This is always relative to "now" which is set when you refresh. So, if you refresh at 2:30 p.m. you would see all trips with a StopDateTime from 2:30p.m. to 3:00 p.m.
